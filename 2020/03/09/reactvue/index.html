JSX文件类型：融合了js与html两种语言

  1. html段中颗包含js语法
  2. 在js中处理html段
    Javascript

渲染

  1. 传入ReactDom.render方法进行渲染
  • react元素在创建后是不可变的，更新的唯一方法是创建新元素，然后传入ReactDom.render

组件与依赖

props属性可读、可用、不可修改(单向数据流)

  • 渲染方法
    Javascript
  • 多次渲染
    Javascript
  • 对组件进行提取，细化，提高可复用性
  • 利用组件状态优化
  • 组件状态和生命周期
    状态（state）：组件的私有属性，是局部的，完全受控于当前组件；
    属性（JSX属性）：属性是被传入的，只可读不可修改的；
    生命周期：在重新渲染组件前，释放组件所占用的资源（DOM、计时器、eventListener）
    函数组件 ==> 类 ：使组件拥有状态，拥有生命周期

关于setState

  1. 通过对象属性更新，并不会重新渲染
  2. 状态更新可能是异步的
  3. 状态合并更新

事件处理

  1. 事件命名：驼峰命名法
  2. jsx语法：
    传入函数作为事件处理函数，如：onclick={activate}；
    而不是字符串形式，如onclick=”activateLasers()”
  3. 类的监听事件默认不会绑定this，因此添加事件绑定时
    方法1：需要手动bind(this)
    例如：this.handleClick = this.handleClick.bind(this);
    方法2：使用箭头函数
    例如：
    handleClick = () => {
    console.log(‘this is:’, this);
    }

条件渲染

  1. if：判断分支渲染
    && : true && expression 总是返回 expression，而 false && expression 总是返回 false。
  2. 阻止组件渲染： render() { return null } 但不影响生命周期

列表渲染

创建基础列表组件 只需传入列表数组，即可自动生成
使用li标签必须添加一个key值，主要用于在li的一系列标签中唯一标识该元素，用于后续的删除、修改操作。key的作用域是同级节点之间
当数组元素没有确定的id用于标识，可以用index替代

状态提升

目的：用于父组件中使用多个子组件，则将子组件之间的共享状态提升到父组件中进行管理
问题：子组件之间的共享状态state保存到父组件中，子组件通过props读取，但是不能修改；
解决途径：为了使子组件能够修改提升后的共享状态，将父组件的方法作为props属性值传入子组件来实现；子组件借用父组件的方法来修改父组件的state，“虽然很麻烦，但是有助于查找bug”
